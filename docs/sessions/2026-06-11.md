# Session narrative: the founding conversation (2026-06-10/11)

Narrative layer - the story of WHY this graph looks the way it does. Nothing
here is enforceable; conclusions live in nodes (cited inline).

It started as a history question: do CASE tools still exist? The trail ran
from Sparx EA and Visual Paradigm to the IBM Rational line, then to the
tool+methodology combinations descended from Andersen's Foundation/Method-1:
RUP and Rational Method Composer, the open EPF Composer and OpenUP, and
finally OMG Essence - whose alphas-with-state-checklists turned out to be the
load-bearing import (ins-0009, dec-0008).

Two context briefs merged in: a change-management architecture for AI-authored
spec graphs (typed edges, event-sourcing hybrid -> dec-0003, dec-0006,
dec-0009) and the System Encyclopedia design (typed artifact graph, maturity
ladders, linter-as-heart). The synthesis: endeavor and content as one graph
(dec-0007), state cards with machine/human-typed items (dec-0008), and a
scheduling loop as the factory engine (com-0006).

Then the owner showed his actual repos, and the diagnosis inverted. The
overwhelming complexity was not the systems being built - it was
self-generated process mass (ins-0004). software-factory v4: 3.2MB of prose
specs reviewed by agent panels, nothing runnable (ins-0005). The k8s-platform
forensics supplied the empirical law: mechanical enforcement ended every bug
class it touched; prose rules ended none (dec-0001). The failed project was
the most valuable one, because it alone touched reality.

The correction that followed mattered: the lesson was never "less structure"
but "no described structure, only enacted structure" (ins-0006, dec-0002).
Structure maximalism, prose minimalism. The owner pushed back twice, and both
objections became architecture: methodology is the design center, not a
hazard to manage around; and a slice without a persistent whole is just an
orphaned MVP - so the whole vision gets captured at L0 first (dec-0005), then
slices climb.

The deeper purpose got named along the way: agents removed the bottleneck
that hid the complexity (ins-0001); the ladder is an entropy dial that stages
ambiguity rather than eliminating it (ins-0002); the system exists to
metabolize uncertainty at a controlled rate while protecting
discovery-through-use (ins-0003). The work had stopped being fun because the
reward loop - clean green from scratch - never fired (ins-0007); sessions are
now designed around a guaranteed witnessed transition.

Finally, mechanics: this repo became the receptacle. Writes flow from Claude
chat through Jentic to the GitHub Git Data API (docs/runbook-chat-writes.md).
The first attempt produced lesson les-0001 - a read against a public repo is
not a write test. Three commits later the graph held its founding 40+ nodes,
verified each time by fresh clone and integrity check (ins-0008: derived
views cannot lie).
